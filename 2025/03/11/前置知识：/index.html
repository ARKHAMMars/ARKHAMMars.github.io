前置知识：

PDDL:

一种用于描述规划问题领域的形式化语言。它是人工智能领域中用于描述规划问题的一种常见语言。PDDL主要用于描述有关规划问题的领域知识，包括问题的初始状态、目标状态、操作符、约束条件等。PDDL通常被用来描述问题的初始状态和目标状态，然后通过规定一系列可用的操作符以及这些操作符对状态的影响来解决问题。
PDDL通常由以下几个部分组成：

  1. 声明（Declarations）：包括定义对象、谓词和函数等。
  2. 初始状态（Initial State）：描述问题的初始状态，即问题开始时各个对象的状态。
  3. 目标（Goals）：描述问题的目标状态，即问题需要达到的状态。
  4. 操作符（Actions）：定义可用的操作符或动作，以及这些操作符对状态的影响。
  5. 约束条件（Constraints）：描述解决问题时需要满足的条件。
    使用PDDL可以帮助研究人员更清晰、准确地描述规划问题的领域知识，有助于计算机系统理解和解决这些问题。

Generalized Planning程序特征

  1. 带有条件和循环的规划/程序
    这是一种较常见的表示方式：最终的广义规划可写成类似伪代码或小型程序的形式，其中包含：

    • if-then-else 条件分支
    • while 或 for 循环
    • 对特定情形的处理策略
      这种形式易于理解并具有较强的表达能力，能够嵌入复杂逻辑，适用于在不同状态下执行不同操作。

  2. 有限状态机（Finite State Machine, FSM）或控制器
    有些研究会将广义策略表示为一个有限状态机。其中每个状态对应某种“决策子策略”或“控制模式”，依据当前环境观测和状态机中的转移条件（如“是否已经搬运完所有箱子”）进行状态跳转，从而完成不同实例下的一系列动作。

  3. 层次化任务网络（HTN）
    在分层规划（Hierarchical Planning）中，可以用HTN（Hierarchical Task Network）来表示更高层次的任务分解与子任务结构，如果能找到对多个实例都适用的一套分解方案（即一套“广义”HTN规划模板），也可以看作是一种广义规划的表达。

  4. 强化学习和策略学习的结合
    在一些更贴近机器学习（尤其是强化学习）的场景下，会把Generalized Planning的思路整合进策略学习当中，希望训练出一个能够处理不同规模、不同初始条件的“政策网络（Policy Network）”或“策略函数”。

    • 这类方法往往需要大量训练数据或模拟环境，用学习到的策略覆盖尽可能多的任务实例。

论文梳理

目的

探究大语言模型是否能作为广义规划器：给定一个领域和训练任务，生成一个程序，该程序能高效地为该领域内的其他任务生成规划。

方法

使用规划域定义语言（PDDL）描述的领域，并利用GPT-4来合成Python程序。

  • 思维链（CoT）总结，即促使大语言模型（LLM）在合成程序之前，先用语言总结该领域并提出策略
  • 自动调试，即根据训练任务对程序进行验证，若程序出现错误，会依据四种反馈类型重新向大语言模型发出提示。

Related Work

“LLMs for (PDDL) Planning.”

  1. 最相近的为“Valmeekam et al. (2022); Silver et al. (2022)”的使用LLMs来在PDDL上规划，但广义规划在多任务下成本高，以及由窗口大小限制。

  2. 微调大模型来解决PDDL任务

  3. 其他近期的工作则考虑使用大语言模型在自然语言和 PDDL 之间进行翻译

“Generalized Planning”

  1. 在广义策略的假设类中进行搜索
  2. 使用示例规划来构建广义规划，通常用有限状态机表示
  3. 发现状态和动作抽象，然后将其用于广义规划

PG3通过使用候选广义规划（表示为提升决策列表目标条件策略）来约束示例规划的生成，从而应对这一挑战。

用PG3当作主要对比实验。

“LLMs for Code Generation”

  • 方法基于最新的技术将LLM用于代码生成
  • 参考了通过人为失败验证检查的反馈重新训练LLM。

除了大模型生成程序，还可以用程序综合和归纳逻辑编程来生成代码

背景和问题设置

PDDL域和任务

数据集：

  • 使用带有类型和负前提条件的 STRIPS 子集
  • 对 PDDL 进行非正式描述

PDDL域：

名称、一组类型、一组谓词和一组操作符来定义

PDDL域中的广义规划

在训练阶段，会从这些任务分布中选取一小部分训练任务（在实验中通常为 10 个或更少）。而一组预留的评估任务（通常涉及更多的对象）则用于衡量性能。

目标：

  • 利用训练任务来合成一个程序，该程序要能为所有评估任务生成有效的规划方案。
  • 程序能在固定的时间预算内（实验中为 30 秒）为评估任务返回一个有效的规划，认为这个评估任务得到了解决。
  • 关注的是能够满足要求的规划，而非最优规划

自动交互式调试

利用训练任务来验证这一实现方案：

执行获取规划的操作，直到它返回一个输出结果、抛出异常或者达到超时时间（30 秒）。

  • 输出的是一个有效的规划，我们就继续处理下一个训练任务。
  • 否则，我们会根据以下四种反馈类型中的一种来重新给出提示

“Python Exceptions.”

如果执行 “获取规划（get plan）” 操作导致了 Python 异常，我们会捕获追溯信息，并将其与输入内容一起反馈给大语言模型（LLM）

“Timeout.”

如果 “获取规划（get plan）” 操作在超时时间之前未能完成，我们会告知大语言模型（LLM）该程序未完成，并提示可能存在无限循环的问题。

“Plan Syntax.”

如果 “获取规划（get plan）” 操作返回了一个输出结果，我们会检查其语法：检查它是否是一个字符串列表，每个字符串是否都包含在括号内且以空格分隔，以及每个操作的操作名称、对象名称和每个操作对应的对象数量，相对于该领域和任务而言是否有效。

“Plan Semantics.”

用 VAL 工具（豪伊、朗和福克斯，2004 年）来检查 “获取规划” 的输出是否是一个语义上有效的规划。

论文精读

科学问题

相关方法及其局限性：

  1. ​传统广义规划的局限性

    • ​缺陷1：策略表示能力受限
      经典广义规划方法（如PG3）依赖人工设计的符号化策略模板（如决策列表、有限状态机），难以表达复杂领域逻辑（如递归操作、动态对象集合遍历）。

    • ​缺陷2：搜索效率低下
      策略空间的组合爆炸导致传统方法无法处理大规模任务（如超过100个对象的物流配送问题），时间复杂度随对象数量呈指数增长。

  2. ​LLM直接用于规划的不足

    • ​缺陷3：生成结果不可控
      LLM直接生成计划序列时，可能违反PDDL语义约束（如忽略操作前提条件），导致无效计划。

    • ​缺陷4：逐任务规划的重复性
      LLM为每个任务独立生成计划，无法复用领域知识，导致计算冗余（如相同子问题重复求解）。

  3. ​符号化规划与学习的割裂

    • ​缺陷5：符号方法的僵化性
      纯符号化规划（如PDDL）依赖人工编码领域知识，难以自适应新场景（如动态增减配送节点）。

    • ​缺陷6：学习方法的黑盒性
      基于强化学习（RL）的方法缺乏可解释性，难以验证其策略是否符合领域逻辑。

  4. ​验证方法的不足

    • ​缺陷7：人工验证不可靠
      传统广义规划依赖人工测试有限任务实例，无法保证生成策略的通用性。

鉴于上述方法的缺陷，故而结合LLM和广义规划来生成领域内的广义规划。

核心问题：

1.如何引导LLM正确理解特定领域内的设定，不发生差错

2.如何引导LLM生成复合PDDL文法的程序结构

3.如何确保LLM生成的代码符合问题逻辑

4.如何引导LLM进行自主的debugging环节

5.面对复杂问题和LLM窗口大小有限的冲突问题如何进行处理

6.验证各个组成部分对于结果的影响，寻找重要的组成部分

解决方案

1.使用PDDL域（Planning Domain Definition Language）​作为输入优势在于形式化表达能力、与经典规划系统的兼容性，以及促进大型语言模型（LLM）的领域知识迁移

形式化表达与结构化知识传递：

通过谓词逻辑（Predicates）​和动作模式（Actions）​明确定义领域内的状态、操作前提与效果。这种形式化特性使得LLM能够精确理解任务逻辑，而非依赖模糊的自然语言描述。一定程度上解决了问题1.

与经典规划系统的兼容性
经典符号规划器的标准输入格式，将其作为LLM的输入可实现混合架构协同：

  • 有利于执行验证与调试支持，可以从语法上进行检查
  • 使用PDDL能够直接和PG3的对比，验证优势

提升LLM的领域泛化能力

可聚焦于策略抽象而非底层搜索细节，从而突破传统规划的局限性

  • 抑制搜索倾向

  • 对象无关性适配：PDDL的变量参数化（如?block）使LLM生成的程序能处理任意规模任务（如3块或1000块积木的堆叠问题）

支持自动调试与反馈闭环

结构化错误信息（如未满足的前提条件）为LLM提供精准调试依据

  • 错误类型分类：可直接明确分为4种

  • 循环不变式提取：支持形式化验证工具（如定理证明器）提取循环不变式，确保生成程序的通用性

标准化与可扩展性优势

PDDL的广泛应用使其成为连接LLM与规划领域的理想桥梁：

  • 领域扩展便捷：新增领域仅需提供PDDL描述，无需重构模型架构（如自动驾驶、机器人导航等场景）

  • 研究可复现性：PDDL的公开基准库（如IPC竞赛领域）便于其他研究者复现实验，验证方法的普适性

对比喻其他的**基于指针的编程语言表示**等输入方式，整体更加有利于自动化的形成我们所需要的标准化、兼容符号规划器

使用CoT的思想，形成了合理的**混合架构的框架**
